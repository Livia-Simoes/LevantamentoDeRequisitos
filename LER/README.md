- O que é Levantamento de Requisitos 
    - Transformar as necessidades do negócio em infomações técnicas

- Requisitos Funcionais e Não Funcionais 
  - RF - o que o sistema vai fazer
 - RNF - como o sistema vai fazer

- Requisitos, Regras, Restrições 
    - Requisitos: a parte tecnica do sistema. 
    - Regras: as necessidade dos clientes.
    - Restriçõs: Limitações impostos pela empresa, linguagem, leis, normas.

### Técnicas de levantamento de requisitos
- briefing
- brainstorm
- questionário
- entrevista
- etnografia
- workshop
- prototipagem

#### briefing
- coleta de dados iniciais
- não existe um formato pré-definido
- permite coleta de diversas informações do cliente
  - Manual de identidade visual (cores ,logo ,slogan , fontes etc.)
  - concorrentes
  - prazos
  - orçamento
  - identidade da empresa (visão. missão e valores)
  - público-alvo

#### brainstorm
- reunião criativa com objetivo de solucionar um ou mais **problemas**
- podem ser estruturadas ou não estruturadas 
- reunião curta com foco bem definido 
- ferramentas de brainstorm
  - Miro
  - Mentimeter
  - OneNote

#### questionário
- perguntas direcionadas ao cliente, com objetivo de montar o escopo do projeto (documentação final do projeto)
- formas de fazer o questionário
  - reunião com perguntas diretas (qualitativas)
  - formulário de coleta de dados (quantitativas)

#### entrevistas
- instrumento de coleta de dados com usuário do projeto 
- entrevista podem ser formais informais 
- nessa etapa é necessário um MVP (produto minimo viável)

#### etnografia
- observação da cultura da empresa, a forma como ela atende e se relaciona com o seu publico interno e externo.

#### workshop
- técnica de levantamento de requisitos que acontecem na apresentação de uma mvp, para um grupo seleto de pessoas (stakeholders) com intuito de absorver a reação do publico ao produto
- eventos, feira, workshops de produtos especificos

#### prototipagem
- uma versão preliminar, simples e funcional, sendo uma simulação do produto final, com intuito de testar a funcionalidade antes de começar a escrever código 
- os prototipos podem ser de 
  - baixa fidelidade (rafe): rascunho de um desenho feito a mão
  - média fidelidade (rough): desenhos no computador para marcação de espaços e layouts, não precisa de texto nem cores
  - alta fidelidade: prototipo focado nas aplicações de funcionalidade iguais aos do projeto que será desenvolvido posteriormente em código.
